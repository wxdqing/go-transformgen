# transformgen example

This example verifies the Go request/response/notify flow generated by `transformgen`.

- `example/transform`: proto files, YAML definitions, generated protobuf code, and generated protocol code.
- `example/demo`: business modules that simulate a real fx-bootstrap application. `player`, `battle`, and `chat` each expose a `HandlerModule` through an fx group.

## Regenerate

From `tools/source/transformgen`:

```bash
protoc \
  --go_out=. \
  --go_opt=module=github.com/wxdqing/go-transformgen \
  -I . \
  example/transform/heartbeat.proto \
  example/transform/battle.proto \
  example/transform/chat.proto

protoc \
  --descriptor_set_out=example/transform/transform.pbset \
  --include_imports \
  --include_source_info \
  -I . \
  example/transform/heartbeat.proto \
  example/transform/battle.proto \
  example/transform/chat.proto

go run ./cmd/transformgen \
  --proto-set example/transform/transform.pbset \
  --defines-dir example/transform/defines \
  --target go \
  --side requester,responder \
  --runtime emit \
  --out example/transform/protocol \
  --package protocolpb
```

Generate Go requester-only helpers:

```bash
go run ./cmd/transformgen \
  --proto-set example/transform/transform.pbset \
  --defines-dir example/transform/defines \
  --target go \
  --side requester \
  --runtime emit \
  --out example/transform/protocol_requester \
  --package protocolpb
```

Generate C# protocol helpers:

```bash
go run ./cmd/transformgen \
  --proto-set example/transform/transform.pbset \
  --defines-dir example/transform/defines \
  --target csharp \
  --side requester,responder \
  --runtime emit \
  --out example/transform/csharp \
  --package Plan.Protocol
```

`*.pbset` files are ignored and can be regenerated at any time.

Run the demo:

```bash
GOWORK=off go test ./example/...
```
